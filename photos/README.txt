PRAIRIE BLOOM PLANNER  -  PLANT PHOTO / ICON FOLDER
======================================================

HOW IT WORKS
------------
Keep this 'photos' folder in the SAME place as prairie-bloom-planner.html
(i.e. side by side). When you open the planner, each plant card looks for an
image in here named after that species. If it finds one, it shows it as a banner
photo across the top of that plant's card; if not, the card just shows the slim
colour strip (and the colour dot by the name) as before.
Nothing else is required - no internet, no setup.

FILE NAMING
-----------
Name each file with the species id below, plus an image extension.
Accepted extensions (tried in this order):  .jpg  .jpeg  .png  .webp
Example:  photos/crocus.jpg   or   photos/redlily.png

TIPS
----
* Wide / landscape images look best (the banner is cropped to fill). ~600x300 px is plenty.
* Keep files small (under ~150 KB each) so the planner stays quick to open.
* You can add only the ones you have - missing plants fall back to the colour dot.


PHOTO CREDITS / ATTRIBUTION
---------------------------
Attributions are managed in this folder, in the file:  credits.js
Open it in any text editor. For each photo you add, find that species' line,
delete the leading // , and fill in the photographer, licence and source URL.
The planner shows the credit as a small caption in the corner of that photo's
banner (and links to the source if you give a URL). Leaving a line commented
out, or deleting credits.js entirely, simply means no credit is shown.
Most CC-licensed images (e.g. from Wikimedia Commons) require crediting the
author and naming the licence — credits.js is where you record that.

WHERE TO FIND OPEN-SOURCE PHOTOS
--------------------------------
Each card also has a 'Photos' link that opens Wikimedia Commons for that species,
filtered to freely-licensed images. Most are CC-BY / CC-BY-SA / public domain -
free to reuse, usually with attribution. Always check the licence on the image
page. iNaturalist (the 'ID guide' link) is also good for identification photos,
but licences there vary per photo. Your own photos are, of course, ideal.
